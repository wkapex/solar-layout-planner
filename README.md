# 野立て太陽光 配置平面図 作成ツール

公図 / 設計会社の敷地平面図 / Googleマップ航空写真（JPG・PNG・BMP・PDF）を読み込み、
野立て（地上設置）太陽光パネルの配置平面図を自動生成して **DXF（CAD）** で出力するブラウザツールです。
サーバー不要・ローカル完結で動作します。

## 使い方

```bash
npm install
npm run dev      # ブラウザで http://localhost:5173 が開く
npm run build    # dist/ に静的ファイルを出力（配布用）
```

### 操作の流れ（画面左のステップ）

1. **敷地図を読み込む** — PDF / JPG / PNG / BMP をドロップ選択。ホイールでズーム、右ドラッグ（またはShift+左ドラッグ）で移動。
2. **縮尺を校正** — 2方式を選べます。
   - **実距離で校正**（航空写真・設計図向き）：実距離が分かる2点をクリック → 実距離(m/km)を入力 →「この距離で校正」。航空写真は「距離を測定」の辺の数値をそのまま入力。
   - **公図座標で校正**（公図向き・推奨）：現地調査前で実距離が無くても校正できます。公図の四隅に印字された**X(北)・Y(東)座標**を持つ基準点を2つ（右上・左下推奨）クリックし、各点のX・Y座標と**座標系番号(1〜19系)**を入力 →「公図座標で校正」。
   - ※**DXF出力はミリメートル単位**（$INSUNITS=mm）。JWCAD・AutoCAD で原寸（例：20m＝20000）で開けます。印刷時は用紙スケール（1/500 等）を指定してください。
     - **縮尺・北方向・緯度経度を自動算出**します（北方向は座標軸から、緯度経度は座標系番号の原点から逆算）。住所入力は不要になります。
     - 北方向は「6. 制約」の北の向きへ、緯度経度は「4. 設置場所」へ自動反映されます。
3. **敷地を多角形で囲む** — 「作図開始」→ 頂点をクリック →「閉じる」。面積(m²)が表示されます。
4. **設置場所** — 住所を入力して緯度経度を取得（国土地理院API）。取得できない場合は緯度経度を直接入力。
   - 太陽高度の計算に使います。
5. **パネル仕様（手動入力）** — メーカー・型式・幅(mm)・高さ(mm)・出力(W)。「履歴に保存」で次回再利用できます。
6. **配置の制約**
   - **設置タイプ**：
     - *野立て（地上設置）*：傾斜角＋冬至の影離隔で配置。A=真南／B=敷地なりの2パターン。
     - *陸屋根（フラット屋根）*：**合掌（東西・背中合わせ）の山型配置**。1山＝片側1枚×2スロープ。横の列間＝20mm（「列内ギャップ」）、山と山の離隔＝固定値（既定250mm）。低傾斜・東西向きのため冬至影離隔は使わず密に配置。住所・段数は不要、傾斜角は合掌の傾き（奥行投影）に使用。屋根なり1パターン。
     - *傾斜屋根（パネルを寝かせてフラッシュ設置）*：**常に横置き**・影離隔なしの**均一グリッド**で、東西の列が全行で揃う**最大矩形**を自動配置（ガタつき防止）。屋根なり1パターン。傾斜角・段数・パネル向き・列数モード・上部の境界離隔は使いません。横間隔＝列内ギャップ、縦間隔＝行間ギャップ、**屋根端離隔は東西・南北を個別指定**（防火・点検の離隔を方向別に設定可）。自動最大の後、**縦(段数)・横(列数)を直接入力すると枚数を調整可能**（同じ向きのまま縮小）。屋根面は1面ずつ実行。
   - 向き（縦置き/横置き）、段数
   - **列数モード**：
     - *最大列数*：上限内で端数列を許容して詰める
     - *指定列数*：結線・PCS直列数に合わせ、列数を最大3パターン指定（例 12/8/4）。指定列数のアレイのみで配置し、**列数の大きいものから優先**配置します。
   - 傾斜角・境界離隔・列内ギャップ・アレイ間横ギャップ
   - 北の向き（画像の上方向から時計回り。航空写真で北が上なら 0）
   - 前後ピッチ（空欄＝自動計算、数値を入れると上書き）
7. **配置を生成** — 2パターンを生成して比較表示。表示切替で Canvas プレビューが切り替わります。
   - **パターンA：真南設置優先**（全アレイを真南向き）
   - **パターンB：敷地なり最大枚数**（敷地の辺方向に沿わせて枚数最大化。真南からの振れ角を表示・図面注記）
8. **DXF出力** — パターンを選んでダウンロード。レイヤ：`SITE`(敷地線) / `PANEL`(パネル) / `NOTE`(方位・注記)。

## 影離隔（前後ピッチ）の自動計算

設置場所の緯度経度から、**冬至の 10:00〜14:00（JST）** の太陽高度・方位を求め（`suncalc`）、
その時間帯で前後アレイへ影が落ちない最小間隔を自動算出します。
北海道のように冬の太陽高度が低い地域ほどピッチは自動的に大きくなります。

## 注意・制約（MVP）

- 自動配置は**机上の最大設置可能枚数の目安**です。電気設備スペース・造成・詳細離隔で実際は減ります。
- 縮尺校正の精度が図面全体の精度を決めます（特に公図）。基準点は四隅の座標目盛り位置を正確にクリックしてください（2点が離れているほど精度が安定）。
- 公図座標からの緯度経度は太陽高度計算が目的のため球面近似です（系内で誤差 <0.05°、太陽角には十分）。厳密な測量用途ではありません。
- 敷地多角形は凸〜単純形状を想定（4隅＋中心の内外・離隔判定）。複雑な凹形状では端部が過大評価される場合があります。
- DXFのTEXT注記は文字化け回避のため方位を英字で併記しています（例：`AZIMUTH: 30deg EAST of SOUTH`）。
- 概算年間発電量・経済性は出力しません（意図的に非対象）。
- 周辺建物・地形による影は未考慮（アレイ間の相互影のみ）。
- 傾斜屋根（フラッシュ）は屋根外形を「パネル設置面の実寸」として扱い、屋根勾配による平面投影の縮小は考慮しません（概算用の簡易化）。複数の屋根面は1面ずつ実行してください。

## 構成

```
src/
  imageLoader.ts  PDF/画像の読込
  canvas.ts       表示・ズーム/パン・オーバーレイ
  calibration.ts  縮尺校正（2点距離→m/px ／ 公図座標→m/px・北方向）
  jpr.ts          平面直角座標系（座標系番号→原点、座標→緯度経度）
  polygon.ts      座標変換・面積
  geometry.ts     幾何ユーティリティ（内外判定・離隔・射影）
  panels.ts       パネル手動入力＋履歴
  geocode.ts      住所→緯度経度（GSI）
  solar.ts        冬至10-14時の太陽位置
  layout.ts       自動配置（パターンA/B・列数モード）
  summary.ts      集計表示
  dxfExport.ts    DXF書き出し
  main.ts         UI統合
```
